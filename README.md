# 資材移動管理

運送会社のドライバーがWebフォームを開き、
**自分の車両に積んでいる資材の数**を報告するアプリ。
報告はGoogleスプレッドシートに記録され、**車両を並べたダッシュボード**で
資材数をリアルタイムに確認できる。

```
入力フォーム → GAS → スプレッドシート「記録」
                                                    ├ 「最新」シート（車両×資材の最新表）
事務所のPC/モニター ← docs/dashboard.html ←─ GAS(doGet JSON)
```

## 構成

| ファイル | 役割 |
|---|---|
| `docs/index.html` | ドライバー用フォーム（車両選択＋資材ごとの数量入力） |
| `docs/dashboard.html` | 車両を並べたリアルタイムダッシュボード（30秒ごと自動更新） |
| `docs/config.js` | 設定（GAS URL・車両リスト・資材リストと定数） |
| `gas/Code.gs` | 受信サーバー（Google Apps Script） |

## ダッシュボードの見方

- 車両ごとにカードが並び、資材ごとの最新報告数を表示
- 車両カードを押すと、その車両を選択済みの報告フォームが別タブで開く
- 色分け：🟥 0個 ／ 🟨 定数未満（不足） ／ 🟩 定数以上
- 当日報告がない車両は「⚠ 昨日」など古さを警告表示、未報告はグレー

## セットアップ手順

### 1. スプレッドシート + GAS（受信側）

1. [Googleスプレッドシート](https://sheets.new)を新規作成（名前は「資材報告」など）
2. 拡張機能 > Apps Script → `gas/Code.gs` の中身を貼り付けて保存
3. 関数「`setup`」を選んで実行（初回は権限の承認が必要）→「記録」「最新」シートができる
4. デプロイ > 新しいデプロイ > 種類「ウェブアプリ」
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
5. 発行された `https://script.google.com/macros/s/…/exec` を `docs/config.js` の `GAS_URL` に貼る

### 2. config.js の調整

- `VEHICLES`：実際の車両名に書き換える
- `ITEMS`：資材名と定数（積んでおくべき数）に書き換える

### 3. GitHub Pages（フォーム・ダッシュボードの公開）

1. このリポジトリをGitHubにpush
2. Settings > Pages > Branch: `main` / フォルダ: `/docs` で公開
3. フォーム：`https://<ユーザー名>.github.io/<リポジトリ名>/`
   ダッシュボード：`https://<ユーザー名>.github.io/<リポジトリ名>/dashboard.html`

### 4. 利用開始

- GitHub PagesのフォームURLをLINEなどで共有
- 一度入力した報告者名は次回から初期表示される

### 5. 資材数の確認方法（どちらでも）

- **ダッシュボード**：`dashboard.html` のURLを開くだけ。事務所のモニターに
  出しっぱなしにすれば30秒ごとに自動更新。Googleサイトに「埋め込み > URL」でも貼れる
- **スプレッドシート**：「最新」シート（車両×資材の表）をGoogleサイトに
  挿入 > スプレッドシート で埋め込み

## 動作確認

- `GAS_URL` が空 → ダッシュボードがデモデータを表示
